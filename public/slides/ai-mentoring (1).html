다음 토큰 예측

LLM은 입력된 텍스트를 바탕으로 가장 그럴듯한 다음 단어를 확률적으로 선택하는 모델이다. "이해"가 아니라 패턴 예측이다.

토큰(Token)

텍스트를 쪼개는 최소 단위. 영어 단어 1개 ≈ 1.3토큰, 한국어는 글자당 2~3토큰. 컨텍스트 윈도우: 한 번에 처리할 수 있는 텍스트의 최대 길이다.

파라미터

모델을 구성하는 가중치의 수. GPT-4는 추정 1조+, Llama 3는 700억. 크다고 무조건 좋은 건 아니고 용도에 맞는 모델이 중요하다.

핵심 요점: LLM은 "정답을 알아서" 출력하는 게 아니다. 학습 데이터 패턴으로 가장 그럴듯한 텍스트를 생성한다. 그래서 자신 있게 틀린다.

프롬프트는
AI와의 인터페이스다

코드는 컴퓨터에게 명령하는 언어다.
프롬프트는 AI에게 명령하는 언어다.

같은 AI, 같은 모델이라도 프롬프트를 어떻게 쓰느냐에 따라 결과물의 품질이 완전히 달라진다. 프롬프트는 운이 아니라 기술이다.

프롬프트를 대충 쓰면

"뭔가 알려줘" → 피상적인 설명, 쓸모없는 예시, 재질문 필요

프롬프트를 잘 쓰면

"내가 원하는 것을 정확히" → 바로 쓸 수 있는 코드, 문서, 답변

실제 사례 — 한국사 과제

같은 AI, 다른 결과

과제: "조선 후기 사회 변화를 서술하시오"

✗ 프롬프트 없이
PROMPT "조선 후기 사회 변화 알려줘"
RESULT 신분제·상공업 나열. 흐름 없음.
그대로 못 씀 → 손으로 다시 작성
낮은 점수
✓ 프롬프트 설계
PROMPT "고교 2학년 한국사 서술형 과제야.
① 신분제 동요 ② 상품 경제 발달
③ 서민 문화 성장 — 각 항목 사례 1개,
500자로 써줘"
RESULT 구조화된 서술, 사례 포함.
검토·수정만 하면 바로 제출
만점

역할 / 맥락 / 형식 / 분량 — 네 가지만 넣어도 결과가 완전히 달라진다.

프롬프트 작성법

6가지 원칙

  • 01

    역할을 지정하라

    "시니어 백엔드 개발자로서[역할], FastAPI 프로젝트에서 JWT 인증을 추가해야 해[맥락]. 함수 단위로[형식] 핵심 코드만 보여줘[분량]." — 네 요소를 모두 넣으면 바로 쓸 수 있는 답이 나온다.

  • 02

    원하는 출력 형식을 명시하라

    "JSON 형태로", "함수 단위로", "주석 포함해서", "마크다운 없이 코드만" 처럼 형식을 지정한다.

  • 03

    예시를 넣으면 정확도 급상승

    Input/Output 예시를 1~3개 제공하는 Few-shot 프롬프팅. 말로 설명하기 어려운 패턴에 효과적이다.

  • 04

    한 번에 너무 많이 요청하지 마라

    복잡한 작업은 단계를 나눠서 요청한다. "먼저 설계를 보여줘 → 승인 후 구현" 식의 대화형 접근이 유리하다.

  • 05

    에러 메시지는 통째로 붙여넣어라

    요약하지 말고 스택 트레이스 전체를 포함한다. AI는 맥락을 잘 볼수록 정확한 진단을 낸다.

  • 06

    마음에 안 들면 즉시 피드백

    "반환값을 Optional로 바꿔줘", "에러 처리 빠진 부분만 추가해줘" 처럼 행동 가능한 피드백으로 반복 개선한다.

실전 활용

AI와 함께 코드 쓰는
실전 패턴

탐색 (Explore)

"Python에서 CSV 파일을 읽는 가장 일반적인 방법 3가지 보여줘"
— 모르는 영역 빠른 조망에 탁월

구현 (Implement)

"JWT 인증 미들웨어를 Express.js로 구현해줘. refresh token 포함, TypeScript 기준"
— 상세 스펙 제공할수록 품질 향상

디버깅 (Debug)

"이 에러가 왜 발생하는지 설명하고 수정해줘: [에러 전문 붙여넣기]"
— 에러 맥락과 코드를 함께 제공

리뷰 (Review)

"이 코드의 보안 취약점, 성능 이슈, 개선점을 시니어 개발자 시각에서 검토해줘"
— 자기 코드 품질 빠른 점검

실습 01 — 설치

Gemini CLI 설치

# Node.js 버전 확인 (18 이상 필요)
node --version

# npm으로 전역 설치
npm install -g @google/gemini-cli

# 설치 확인
gemini --version

# 또는 npx로 설치 없이 바로 실행
npx @google/gemini-cli

설치 후 최초 실행 시 브라우저가 열리며 Google 계정 로그인 및 OAuth 인증이 진행된다. 무료 티어는 분당 요청 제한이 있지만 실습에는 충분하다.

실습 02 — 기본 사용

기본 명령어

# 첫 실행 시 브라우저가 열리며 Google 계정 자동 인증
gemini

# 단일 질문 (파이프 활용)

함정

JPA, MSA, Reactive… 프레임워크 사용법에 3년을 허비하지 말 것.
유행하는 도구보다 어디서든 쓰이는 CS 원리가 더 오래간다.

변하지 않는 본질
  • 빠른 학습 능력 — 새 기술을 빠르게 흡수하는 힘
  • 깊이 있는 CS 지식 — AI 코드의 성능·환각을 잡아내는 눈
  • AI 활용 능력 — 도구를 다루는 것이 이제 기본 스펙
핵심 전략 1

CS 지식이
경쟁력이 되는 이유

예시

멀티플렉싱이라는 개념을 알고 있다면,
Java Virtual Threads가 왜 ForkJoinPool을 쓰는지
문서를 훑어보는 것만으로 바로 이해된다.

핵심

CS는 도구가 바뀌어도 변하지 않는 원리다.
JPA, MSA 같은 프레임워크는 유행을 타지만,
OS·네트워크·자료구조는 어디서든 쓰인다.

CS 지식이 있으면:

  • AI가 짠 코드의 성능 이슈와 환각을 잡아낼 수 있다
  • 새 기술 스택을 AI와 함께 빠르게 흡수할 수 있다
  • 트러블슈팅 경험을 링크드인에 공유하면 차별화된 포트폴리오가 된다
  • 프레임워크 선택의 트레이드오프를 설명할 수 있다
핵심 전략 2

포트폴리오를
채우는 법

① CS 기반 트러블슈팅

JPA 숨겨진 기능 발견?은 재미없다. OS·네트워크·DB 이론으로 근본 원인을 분석하고 링크드인에 공유하라. 너 빼고 다 아는 얘기는 NO.

② 협업 경험 증명

프로젝트 완성보다 협업 과정이 귀하다. API 명세를 프론트엔드와 함께 설계하고, 소통·조율·피드백 경험을 남겨라. "내가 문화를 바꿨다"는 신입에겐 마이너스.

③ AI 활용 능력 증명

에이전트 만들기, AI 도구로 실제 문제 해결하기 — AI를 쓰는 방법을 아는 것 자체가 포트폴리오가 된다.

④ 오버 엔지니어링 금지

Spring Boot + Kotlin + JPA + MySQL을 당연하게 쓰지 말 것. 왜 이 기술을 선택했는지, 트레이드오프를 설명할 수 있어야 한다. 토끼굴에 빠지지 말 것.
