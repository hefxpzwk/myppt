01 / 08 Service Intro

Philosopher

철학자들의 사상, 문체, 문제의식을 깊이 이해한 AI와 실제로 대화하는 철학 인터페이스입니다.

핵심 목표는 정답 제공이 아니라, 질문과 반박을 통해 사용자의 사고를 더 선명하게 만드는 것입니다.

Socrates
Nietzsche
Plato
Simone de Beauvoir
02 / 08 Problem

우리가 해결하는 문제

  • 생각을 깊게 만들 질문을 스스로 설계하기 어렵습니다.
  • 기존 AI 대화는 한 명의 관점으로 답이 고정되기 쉽습니다.
  • 대화 기록이 길어질수록 맥락 유지와 재사용이 어렵습니다.
03 / 08 Core Value

서비스 핵심 가치

철학자 페르소나

소크라테스, 칸트, 공자 등 9명의 고유한 대화 톤을 제공합니다.

사고 확장 대화

질문-반박-재구성 흐름으로 사용자의 관점을 단계적으로 확장합니다.

텍스트 + 음성

일반 채팅과 음성 모드를 모두 지원해 상황에 맞게 인터랙션할 수 있습니다.

04 / 08 User Flow

사용 흐름

1. 로그인

Google OAuth + Supabase 세션으로 사용자 인증

2. 철학자 선택

이름/시대/학파 검색 후 원하는 철학자와 새 대화 시작

3. 대화 진행

메시지 전송, 답변 음성 재생(TTS), 음성 입력(STT) 지원

4. 프로젝트 관리

대화를 프로젝트로 묶고 이동/고정/삭제/설정 저장

05 / 08 Key Features

핵심 기능 상세

  • 대화 제목 자동 생성: 첫 메시지 기반으로 최대 80자 자동 제목 설정
  • 프로젝트별 지침(instruction) 설정: 철학자 응답 스타일을 컨텍스트 단위로 조정
  • 채팅 운영 기능: 프로젝트 이동, 최근/프로젝트 분리, 핀 고정, 삭제
  • 음성 기능: `/api/v1/tts` 기반 철학자별 음성 출력 + 브라우저 음성 인식 입력
06 / 08 Architecture

기술 구조

Client

Next.js 16 + React 19 + TypeScript + Tailwind CSS v4

Server

FastAPI 기반 `/api/v1` API, 인증 토큰 기반 사용자 데이터 분리

Data/Auth

Supabase Auth(OAuth), 프로젝트/대화/메시지 단위 저장 구조

07 / 08 API Scope

백엔드 API 범위

  • 인증 사용자 정보: `GET /api/v1/me`
  • 프로젝트 관리: 생성/목록/설정수정/삭제
  • 대화 관리: 생성/목록/이동/삭제 + 메시지 송수신
  • 음성 합성: `POST /api/v1/tts` (철학자 voice 매핑, mp3 반환)
08 / 08 Closing

정리

  • Philosopher는 철학자 페르소나 기반으로 사고를 확장하는 대화형 서비스입니다.
  • 텍스트와 음성을 함께 제공해 학습/토론/개인 성찰 모두에 적용 가능합니다.
  • 다음 단계는 실제 사용자 실험을 통해 대화 품질과 학습 효과를 정량화하는 것입니다.
